        API key

        Generate a secret key to connect your own software to our API — push cars, clients and branches in, and pull reservation & rental reports out, and open the interactive Swagger docs below. You must add at least one car, client, or branch before a key can be generated.

        How to authenticate

        Send your key on every request as a header: Authorization: Bearer <key> (X-API-Key: <key> also works). The key is shown once — store it safely; regenerating replaces the old one.

        All available endpoints

        Base URL is this site's address (for local testing http://localhost:5000). For an interactive reference where you can authorize with your key and try every endpoint live, open the Swagger API docs ↗.

        Open interactive API docs (Swagger) ↗
        PurposeMethod & RouteBody / Notes
        Add cars POST /api/cars/batch [ {car}, … ] or {"cars":[…]}
        Add clients POST /api/clients/batch [ {client}, … ] or {"clients":[…]}
        Add branches POST /api/branches/batch [ {branch}, … ] or {"branches":[…]}
        Add everything at once POST /api/batch {"cars":[…], "clients":[…], "branches":[…]} — all-or-nothing
        Fleet report GET /api/reports/cars Every car with its branch & GPS; optional ?branch_id=<id|main>
        Reservation report GET /api/reports/reservations Every reservation with client, car, branch, dates & status
        Rental report GET /api/reports/rentals Every rental with car branch & return branch; optional ?client_id=<id>
        B2B rental report GET /api/reports/company-rentals Cars rented out to companies — with return status & branch
        Note: the report endpoints return data only after your company has records in the system — an empty company gets a clear error instead of an empty report. Up to 500 records per write request.
